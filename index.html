Origin Story

Meet Stephen Wolfram , the man who slaps his last name on pretty much every product he makes, which is entirely excusable given the sheer awesomeness of said products.

He is incredibly smart, having published his first scientific paper at age 15. He dropped out of Oxford at 17 and proceded to receive a PhD in theoretical physics from Caltech by age 20.

While working out new physics equations, he decided to create a program to automate the work, and thus he and Chris A. Cole made Symbolic Manipulation Program - a precursor to Wolfram Mathematica.

So What is Mathematica?

The world's definitive system for modern technical computing.

From Mathematica's website

Technical computing is the application of the mathematical and computational principles of scientific computing to solve practical problems

Mathematica is built on the Wolfram Language, which is based on the following principles:

  • Build as much knowledge into the language as possible
  • Automate as much as possible, and let programmers concentrate on what they want to do (there are approximately 5,000 built-in functions )
  • Maximize readability and understandability
  • Make everything a symbolic expression

Symbols Are Awesome!

Symbolic expressions make possible:

  • Incremental programming: any fragment of code is instantly runnable
  • Any form of data can immediately be represented within the system
  • Code can involve any kind of object, such as an image, document, etc.
  • Easy integration with Wolfram Cloud!

A neat example

The following code generates a list of all hyperlinks on https://www.google.com:

Import["https://www.google.com", "Hyperlinks"]

Elegant, right? Now, this slightly more complicated code will go to each of those links and get all the hyperlinks from those new pages, and then it will generate a graph of all the resulting hyperlink connections:

Graph[Flatten[Rest[NestList[Union[Flatten[Thread[# -> Import[#, "Hyperlinks"]] & /@ Last /@ #]] &, {"" -> "https://google.com"}, 2]]], ImageSize -> Large]

One Last Example

The following function generates a fractal tree, taking as its arguments the coordinates of the base, the angle of the base relative to the horizontal, the length of each branch, the number of iterations, and a paramter "a" that determines the angle between branches:

Tree[p1_, theta_, r_, i_, a_] := Module[{p2 = p1 + r {Cos[theta], Sin[theta]}, graphics}, graphics = {Thickness[0.07 r], RGBColor[0.5 r, 1 - r, 0], Line[{p1, p2}]}; If[i > 1, graphics = Join[graphics, {Tree[p2, theta - a*Pi/6, 0.8 r, i - 1, a], Tree[p2, theta + a*Pi/4, 0.7 r, i - 1, a]}]]; graphics];


Then, the following code generates the cloud app below it:

EmbedCode[ CloudDeploy[ FormFunction["parameter" -> "Number", Show[Graphics[Tree[{0, 0}, Pi/2, 1.2, 9, #parameter], AspectRatio -> 1, Background -> White, PlotRange -> {{-3, 3}, {0, 6}}]] &, "PNG", ImageSize -> 590, AppearanceRules -> <|"Title" -> "Fractal Tree", "Description" -> "Enter a number between 0 and 1."|>], Permissions -> "Public"]]
